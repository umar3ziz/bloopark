Delivery Exchange Process

First of all we have to confirm Sales or Purchase Order to get the delivery process


We configure which Operation type we need to exchange with if we found an exchange type will take it and if we not we take the operation type for Return


First here we should Validate the delivery to be able to make Exchanges and after validating a new Exchange button will appears and we can only exchange Picking moves with the following rules

  • Only moves related to Sales or Purchases will be exchanges
  • We can not Exchange moves that has already Exchanged before
  • We exchange the same product with another Quantity, UOM and Price
As the following Example we exchange
  • For the product [E-COM07] Large Cabinet
    • we change it with product [E-COM06] Corner Desk Right Sit
    • From 1 Quantity to 2 quantities
    • Subtotal decreased from 320$ to 294$
    • but if we get other product we always generate a new Customer Invoice for it
  • For the product [FURN_0269] Office Chair Black
    • Same product and same quantity but we increase the price from 12.5$ to 20$
    • It will generate a new Customer invoice for the price changes
  • For the product [FURN_6666] Acoustic Bloc Screens
    • Same product and same quantity but we decreased from 2950$ to 2000$
    • It will generate a new Credit Note for the price changes
  • For the product [E-COM10] Pedal Bin
    • We leave it as is
    • No invoices will be generated

A new Exchanged Receipt has been created


A new Exchanged widget will show up in Sales and if we just click a new widget will appears with Exchange details with buttons to view all the (Receipts and invoices/Credit notes)


Same as Purchases but instead of Invoices and Credit notes ( Bills and Refunds) -- and instead of Receipts (Deliveries)
